Chapter 12.3: Inference Using Full Joint Distributions

Understanding how to answer probabilistic questions from a complete table of all possibilities

What you'll learn: In this chapter, we explore the most straightforward (but impractical) way to do probabilistic inference: using a table that lists the probability of every possible combination of events. You'll see how to answer any question by looking up and adding values from this table, and why this simple approach breaks down for real problems.

1. The Full Joint Distribution

A full joint probability distribution is simply a table that shows the probability of every possible combination of events. It's the most complete description of uncertainty we can have.

Think of it like this: Imagine you catalogue every weather scenario for your city. Sunny-and-warm, sunny-and-cold, rainy-and-warm, rainy-and-cold, snowy-and-cold... and you list how likely each is. That table IS your full joint distribution.

The Dentist's Office Example

Let's use our running example with three Boolean variables (true/false):

  • Cavity = true if you have a cavity, false otherwise
  • Toothache = true if you have a toothache, false otherwise
  • Catch = true if the dental probe catches (dentist finds it), false otherwise

With 3 Boolean variables, we need 2 × 2 × 2 = 8 rows in our table, one for every possible combination:

Cavity Toothache Catch Probability
truetruetrue0.108
truetruefalse0.012
truefalsetrue0.072
truefalsefalse0.008
falsetruetrue0.016
falsetruefalse0.064
falsefalsetrue0.144
falsefalsefalse0.576
Critical property: All probabilities in the table sum to exactly 1.0 (0.108 + 0.012 + 0.072 + 0.008 + 0.016 + 0.064 + 0.144 + 0.576 = 1.0). This must be true for any valid probability distribution!

What Does Each Row Mean?

Each row represents one specific scenario. For example:

  • Row 1: You have a cavity (true), feel a toothache (true), and the probe catches (true). This happens with probability 0.108 (about 10.8% of the time).
  • Row 8: You have no cavity (false), no toothache (false), and the probe doesn't catch (false). This is the most likely scenario at 57.6%!

2. Answering Any Query with the Joint Distribution

Here's the beautiful power of the full joint distribution: you can answer ANY question by summing the appropriate entries.

The core idea: If you want to know the probability of some event, add up the probabilities of all the rows that match that event.

Example 1: Simple Query

Question: What is P(Cavity = true)?

We need to sum rows where Cavity is true (rows 1-4):

P(Cavity = true) = 0.108 + 0.012 + 0.072 + 0.008 = 0.2
There's a 20% chance you have a cavity.

Example 2: Disjunction (OR) Query

Question: What is P(Cavity ∨ Toothache)? (Probability that you have a cavity OR a toothache, or both)

We sum all rows where at least one is true. It's easier to use the complement:

P(Cavity ∨ Toothache) = 1 - P(¬Cavity ∧ ¬Toothache)
= 1 - (0.144 + 0.576)
= 1 - 0.72 = 0.28
There's a 28% chance you have either a cavity or toothache.

Example 3: Conjunction (AND) Query

Question: What is P(Cavity ∧ Toothache)? (Probability of both a cavity AND a toothache)

Sum only rows where both are true (rows 1-2):

P(Cavity ∧ Toothache) = 0.108 + 0.012 = 0.12
There's a 12% chance you have both.
The pattern: To answer any question, just look at the table and add up the rows that match what you're asking about. It's like filtering a spreadsheet and summing a column!

3. Marginalization: Summing Out Variables

Marginalization means: "I only care about some variables, so I'll ignore (sum out) the others." You're calculating the probability of one or more variables without caring what happens with the rest.

Think of it like this: You have a table of weather data (Temperature, Humidity, Cloud Coverage, Pressure). If you only care about the relationship between Temperature and Humidity, you marginalize out the other two by lumping together all cases with the same Temperature and Humidity, regardless of clouds or pressure.

The General Formula

P(Y) = Σz P(Y, Z=z)

In English: "The probability of Y equals the sum over all possible values of Z of the joint probability of Y and Z."

Worked Example: P(Cavity)

We already calculated P(Cavity) above by summing rows. Let's think of it differently using marginalization:

We want to ignore (marginalize out) Toothache and Catch.

P(Cavity = true) = Σtoothache, catch P(Cavity=true, Toothache, Catch)

Sum all rows where Cavity is true:
= P(C,T,Ca) + P(C,T,¬Ca) + P(C,¬T,Ca) + P(C,¬T,¬Ca)
= 0.108 + 0.012 + 0.072 + 0.008
= 0.2

Another Example: P(Toothache)

Question: What is the overall probability of having a toothache?

Sum all rows where Toothache = true:
P(Toothache) = 0.108 + 0.012 + 0.016 + 0.064 = 0.2
Also 20%! Interesting coincidence in this example.

Two-Variable Marginalization: P(Cavity, Toothache)

We can marginalize out just one variable. For instance, what's P(Cavity = true AND Toothache = true)?

We sum over all values of Catch:
P(Cavity, Toothache) = P(C,T,Ca) + P(C,T,¬Ca) = 0.108 + 0.012 = 0.12
The key insight: When you marginalize, you're throwing away information about variables you don't care about. You group together scenarios that differ only in those ignored variables and add their probabilities. This is how we go from a table with 8 rows to a table with 4 rows (by marginalizing out one variable), or 2 rows (marginalizing out two).

4. Conditioning: Computing Conditional Probabilities

Conditioning means: "Given that we've observed something, what's the probability of something else?" This is where the full joint distribution becomes extremely useful.

The Conditional Probability Formula

P(X|e) = P(X ∧ e) / P(e)

Probability of X given evidence e equals the joint probability of both, divided by the probability of the evidence.

Intuition: Out of all the worlds where the evidence is true, what fraction also have X true? That's the conditional probability.

Step-by-Step Worked Example: P(Cavity | Toothache)

Question: "I have a toothache. What's the probability I actually have a cavity?"

  1. Identify what we need:
    We need P(Cavity=true | Toothache=true)
    Formula: P(Cavity | Toothache) = P(Cavity ∧ Toothache) / P(Toothache)
  2. Calculate the numerator: P(Cavity ∧ Toothache)
    Sum rows where both are true (rows 1-2):
    P(Cavity ∧ Toothache) = 0.108 + 0.012 = 0.12
  3. Calculate the denominator: P(Toothache)
    Sum all rows where Toothache is true (rows 1-2, 5-6):
    P(Toothache) = 0.108 + 0.012 + 0.016 + 0.064 = 0.2
  4. Divide:
    P(Cavity | Toothache) = 0.12 / 0.2 = 0.6
    So if you have a toothache, there's a 60% chance you have a cavity.

Another Example: P(Cavity | Catch)

Question: "The dentist's probe caught. What's the probability I have a cavity?"

Numerator: P(Cavity ∧ Catch) = 0.108 + 0.072 = 0.18
Denominator: P(Catch) = 0.108 + 0.072 + 0.016 + 0.144 = 0.34
Result: P(Cavity | Catch) = 0.18 / 0.34 ≈ 0.529
About 52.9% chance you have a cavity if the probe catches.

Multiple Evidence: P(Cavity | Toothache ∧ Catch)

Question: "I have a toothache AND the probe caught. What's the probability I have a cavity?"

Numerator: P(Cavity ∧ Toothache ∧ Catch) = 0.108
Denominator: P(Toothache ∧ Catch) = 0.108 + 0.016 = 0.124
Result: P(Cavity | Toothache ∧ Catch) = 0.108 / 0.124 ≈ 0.871
About 87.1% chance! More evidence makes it much more certain.
Why this works: The full joint distribution encodes all the relationships between variables. By dividing the joint probability of X and the evidence by the probability of just the evidence, we isolate the probability of X in the worlds where the evidence is true.

5. The Normalization Trick: Using α

Computing conditional probabilities requires dividing by P(evidence). There's a clever trick to avoid this: compute unnormalized probabilities, then rescale them so they sum to 1. The Greek letter α (alpha) represents this scaling factor.

Why This Matters

Computing P(evidence) requires summing over all combinations of values for all the unobserved variables. For large problems, this can be expensive. But we often don't care about the exact denominator—we just want to know the relative probabilities.

Think of it like this: You're deciding between restaurants. One is 5 stars with 200 reviews, another is 4.8 stars with 100 reviews. You don't need to verify the exact counts—you just need to know the relative quality to make a decision. The normalization trick is similar.

The Method: Step by Step

Example: P(Cavity | Toothache)

Step 1: Compute unnormalized probabilities for each value of Cavity

For Cavity = true:
P(Cavity=T, Toothache=T) = 0.108 + 0.012 = 0.12

For Cavity = false:
P(Cavity=F, Toothache=T) = 0.016 + 0.064 = 0.08

Unnormalized: ⟨0.12, 0.08⟩
Step 2: Normalize by dividing each by their sum

Sum = 0.12 + 0.08 = 0.20
α = 1/0.20 = 5

Normalized: ⟨0.12×5, 0.08×5⟩ = ⟨0.6, 0.4⟩
The notation: We write P(Cavity | Toothache) = α ⟨0.12, 0.08⟩ where α is a normalization constant. It means: "compute these two values, then normalize them." The actual value of α doesn't matter—only that it makes the probabilities sum to 1.

The General Pattern

P(X | e) = α ⟨P(X=x₁, e), P(X=x₂, e), ..., P(X=xₙ, e)⟩

For each possible value of X, compute the joint probability with the evidence, then normalize.

Why Use Normalization?

  • We only need to compute the numerator, not the full denominator
  • For multiple unobserved variables, this saves massive computation
  • The normalized probabilities automatically sum to 1
  • We get the same answer as the direct method
Important: The normalization trick is a computational shortcut. It doesn't change the meaning—you still need to actually compute those joint probabilities. But by deferring the division until the end, you can sometimes save work (especially in more complex inference algorithms).

6. The General Inference Equation

All inference from a full joint distribution comes down to one equation:

P(X|e) = α Σy P(X, e, y)

Probability of query X given evidence e equals α times the sum over unobserved variables y of the joint probability.

Decoding the Notation

P(X|e)

What we want to compute: the probability of the query X given the evidence e.

α

A normalization constant (we don't compute it explicitly, we just normalize at the end).

Σy

Sum over all possible values of the unobserved variables y.

P(X, e, y)

The joint probability of all three: the query, the evidence, and the unobserved variables.

Applied to Our Example

Question: P(Cavity | Toothache = true, Catch = false)?

Identify:
X = Cavity (what we want)
e = Toothache=true, Catch=false (evidence)
y = (nothing left to sum over in this case)

Computation:
P(Cavity | T, ¬Ca) = α P(Cavity, T, ¬Ca)
P(Cavity=T, T, ¬Ca) = 0.012
P(Cavity=F, T, ¬Ca) = 0.064

Normalize:
Sum = 0.012 + 0.064 = 0.076
P(Cavity=T | T, ¬Ca) = 0.012 / 0.076 ≈ 0.158
P(Cavity=F | T, ¬Ca) = 0.064 / 0.076 ≈ 0.842

Why This Equation Works

This equation works because:

  • We sum over the unobserved variables (marginalization) to get the joint probability of query + evidence
  • We divide by P(evidence) implicitly through normalization
  • The division isolates just the cases where the evidence is true
Universal principle: Every inference algorithm in probability—from Bayesian networks to Markov Random Fields to deep learning—ultimately comes down to computing these kinds of sums. This equation is the foundation!

7. The Scalability Problem: Why This Doesn't Scale

The full joint distribution is conceptually perfect. It can answer any query. But there's a massive, practical problem: it grows exponentially.

The Exponential Explosion

For n Boolean variables, the full joint distribution has 2n rows:

Number of Variables Number of Rows Practical?
3 (dentist example) 8 Yes ✓
10 1,024 Yes ✓
20 1,048,576 (~1M) Uncomfortable
30 1,073,741,824 (~1 billion) No ✗
50 1,125,899,906,842,624 (~1015) No ✗
100 1,267,650,600,228,229,401,496,703,205,376 (~1030) No ✗

A Concrete Nightmare

Imagine a medical diagnosis system with just 100 symptoms and conditions (which is tiny for real medicine). That's 2100 ≈ 1030 possible combinations.

Reality check: There are only about 1080 atoms in the observable universe. You'd need to store 2100 numbers, which is 1030 combinations. Your entire computer couldn't hold it. At 1 nanosecond per number, it would take 1014 years to enumerate them all.

Storage and Computation Costs

  • Storage: Each entry is a decimal number. 100 variables = 1030 entries = impossibly large.
  • Computation: Answering even one query requires summing many of these entries.
  • Representation: You can't even write down the probabilities explicitly.
  • Learning: You'd need 2100 data points just to estimate reasonable probabilities—an infinite impossibility.

Why This Matters

This exponential explosion is why we need smarter approaches:

  • Bayesian Networks: Exploit independence to represent distributions compactly.
  • Markov Networks: Use local factors instead of global probabilities.
  • Sampling Methods: Estimate probabilities without computing everything.
  • Variational Inference: Approximate distributions tractably.
  • Deep Learning: Use neural networks to implicitly represent distributions.
The fundamental lesson: The full joint distribution is the foundation of probabilistic reasoning, but it doesn't scale. The entire field of graphical models, approximate inference, and modern machine learning exists because of this limitation. Understanding this problem is key to understanding why we need Bayesian networks, sampling, and other techniques.

Summary

What You've Learned

  • Full Joint Distribution: A table with 2n entries for n Boolean variables, listing the probability of every possible combination. Must sum to 1.
  • Answering Queries: Add up the rows that match your question. For any event, P(event) = sum of matching rows.
  • Marginalization: Ignore unobserved variables by summing them out: P(Y) = Σz P(Y,Z).
  • Conditioning: Compute P(X|e) by dividing: P(X|e) = P(X∧e) / P(e). The numerator is joint probability; the denominator is the probability of the evidence.
  • Normalization: You can compute unnormalized values and normalize at the end using α. This saves computation by deferring division.
  • General Equation: P(X|e) = α Σy P(X, e, y). This is the universal formula for inference.
  • Scalability Problem: Exponential growth (2n entries) makes this approach impractical for real problems. This motivates all advanced inference techniques.

The Big Picture

The full joint distribution is the "brute force" solution to probabilistic inference. It's theoretically perfect: it encodes all relationships, answers any query, and the math is straightforward. But it's practically impossible for realistic problems. This is why the rest of Chapter 12 and beyond introduces clever ways to represent, compute, and approximate probabilities without explicitly enumerating every combination.

Key Terminology

Full Joint Probability Distribution

A table listing P(X₁, X₂, ..., Xₙ) for all possible combinations of variable values. Complete but exponentially large.

Marginalization

Summing over unobserved variables to get the probability of observed variables: P(Y) = Σz P(Y, Z).

Conditioning

Computing conditional probability given evidence: P(X|e) = P(X ∧ e) / P(e). Isolates probability in worlds where evidence is true.

Normalization (α)

Scaling factor that makes probabilities sum to 1. Allows us to compute unnormalized values and rescale at the end.

Joint Probability

Probability of multiple events occurring together: P(A ∧ B). Read from the full table by summing matching rows.

Evidence (e)

The observed variables or facts. Used to condition the probability: P(query | evidence).

Query (X)

The variable(s) we want to find the probability of. What we're trying to infer.

Exponential Growth

The main problem: n Boolean variables require 2ⁿ rows. 100 variables ≈ 10³⁰ entries—computationally infeasible.
