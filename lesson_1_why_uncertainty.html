Chapter 12.1 — Why Do We Need Uncertainty?

The world is messy. Pure logic isn't enough.

The Scenario: You're building an AI dentist assistant

Your AI needs to decide: does this patient have a cavity? It has some information — the patient has a toothache, and the probe catches on the tooth. But it can never be 100% sure. Why?

👁️

Partial Observability

The agent can't see everything. There's hidden information it simply doesn't have access to.

Dentist: You can't see inside the tooth without drilling. X-rays help but aren't perfect.
🎲

Nondeterminism

The same action can lead to different outcomes. The world has randomness built in.

Dentist: The same filling procedure might work perfectly one time and fail another — depends on factors we can't fully control.
🌀

Sheer Complexity

Even if we could observe everything, there are too many factors to track. We're too "lazy" (in a good way) to model it all.

Dentist: Patient's diet, genetics, brushing habits, stress levels, hundreds of other factors all affect cavity risk.

Why can't we just use logic?

In earlier chapters, AI used logical rules like: "If toothache, then cavity." But this has a big problem...

Pure Logic Approach

"Toothache → Cavity"

But not all toothaches are cavities! Could be gum disease, a cracked tooth, or even a sinus infection.

We'd need to list every single exception: "Toothache ∧ ¬GumDisease ∧ ¬Crack ∧ ¬Sinus ∧ ... → Cavity"

This is called the qualification problem — the list of exceptions never ends.

Probability Approach

"P(cavity | toothache) = 0.6"

Given a toothache, there's a 60% chance of cavity. Simple, practical, and accounts for all the exceptions implicitly.

We don't need to list every possible cause — the probability already captures the overall picture.

Probability lets us be precise about our uncertainty.

How does the agent actually decide?

The agent combines two things:

Best action = what's most likely to happen + how much I care about each outcome

This is called decision theory. It says: a rational agent picks the action that maximizes expected utility — the average "goodness" of the outcome, weighted by how likely each outcome is.

Decision Theory = Probability (what might happen) + Utility (how much I care)

Dentist example: Should we drill? Depends on (a) how likely it's a cavity, and (b) how bad it is to drill unnecessarily vs. leaving a cavity untreated.

Key Concepts from Section 12.1

Uncertainty
The agent doesn't know the full state of the world. This is unavoidable in real-world problems.
Qualification Problem
The impossibility of listing every exception to a logical rule. There are always more edge cases.
Degree of Belief
A number between 0 and 1 that represents how confident the agent is. 0 = impossible, 1 = certain.
Decision Theory
Probability (beliefs) + Utility (preferences) = rational decisions. Pick the action with the highest expected utility.
Expected Utility
The average outcome of an action, weighted by how likely each outcome is. The rational agent maximizes this.

Summary

  • Uncertainty is unavoidable — partial observability, nondeterminism, and complexity guarantee it.
  • Pure logic fails — the qualification problem means you can never list all exceptions.
  • Probability captures uncertainty — degrees of belief between 0 and 1 let agents reason precisely.
  • Decision theory combines probability with utility to pick the best action.
  • Expected utility is the core criterion for rational decision-making.
