The Wumpus World Revisited

Chapter 12.7: Quantifying Uncertainty - Making Smart Decisions in a Risky World

Why We're Revisiting the Wumpus World

You may remember the Wumpus World from Chapter 7, where an agent used logic to navigate a dangerous cave. But logical reasoning has a big limitation: it either knows something for certain or it knows nothing.

In this lesson, we're returning to the Wumpus World with a new superpower: probability. Instead of just knowing what's logically possible, we'll be able to rank possibilities by how likely they are. This lets us make smarter decisions even when we're uncertain.

The Central Question: You've explored three squares and found breezes. Three unexplored squares are reachable. Which one is safest? Logic can't tell us. Probability can.

1. What is the Wumpus World?

The Wumpus World is a 4×4 grid cave where an agent explores squares, searching for treasure while avoiding danger.

The Setup

  • The Grid: 4 columns × 4 rows = 16 squares
  • The Agent: Explores one square at a time
  • The Danger: Each square contains a pit with probability 0.2 (20% chance)
  • Safety Rule: Square [1,1] (bottom-left) is always safe
  • The Signal: If an adjacent square has a pit, you feel a breeze in your current square

Why this setup? Pits are like hidden dangers. Breezes are like clues. The agent must reason: "I feel a breeze here, so a nearby square probably has a pit. Which one?"

2. The Current Situation

What We Know

The agent's exploration so far:

  • Visited squares: [1,1], [2,1], [1,2]
  • Breezes detected at: [1,1], [2,1], and [1,2]
  • Certainty: [1,1], [2,1], [1,2] have no pits (we're standing there!)

The Question: Where to Go Next?

From [1,2], three squares are reachable without visiting a known unsafe area:

  • [1,3] — directly above [1,2]
  • [2,2] — to the right of [1,2]
  • [3,1] — diagonally adjacent through [2,1]

The Logic Problem

Using only logical reasoning (like in Chapter 7), the agent could determine: "These three squares might have pits, but it's logically possible that any of them is safe."

That's not very helpful! We need to know which square is most likely to be safe. This is where probability comes in.

3. Setting Up the Probability Model

To answer "Which square is safest?", we need to set up a probability model. This means defining random variables and their relationships.

Random Variables

We'll define two types of random variables for each square:

Pi,j = "Is there a pit at square [i,j]?"

  • Takes values: True or False
  • Prior probability: P(Pi,j = true) = 0.2 (for all squares except [1,1])
  • Each square is independent — finding a pit at [2,2] doesn't change the probability of a pit at [3,3]

Bi,j = "Is there a breeze at square [i,j]?"

  • Takes values: True or False
  • Rule: Bi,j = True if and only if an adjacent square has a pit
  • This relationship is deterministic (completely determined by the pits)

The Full Joint Distribution

Our probability model assigns a probability to every possible configuration of pits and breezes:

P(P₁,₁, P₁,₂, ..., P₄,₄, B₁,₁, B₁,₂, ..., B₄,₄) = P(Breezes | Pits) × ∏ P(Pᵢ,ⱼ)

What does this mean? The probability of a complete world state (all pits and all breezes) equals:

  • P(Breezes | Pits): How likely are the observed breezes, given where the pits are? (This equals 1 if the breezes match the pits, 0 otherwise)
  • ∏ P(Pᵢ,ⱼ): The product of probabilities for each individual pit
Think of it like this: Imagine you're trying to reconstruct a mystery. You have clues (the breezes) and suspects (the pit locations). For each possible arrangement of suspects, you ask: "Does this arrangement explain the clues?" If yes, you compute how likely this arrangement is based on your prior beliefs about suspect locations.

4. The Naive Approach: Enumerate Everything

In theory, we can solve this problem by enumerating all possible pit configurations and checking which are consistent with the observed breezes.

How Many Configurations Are There?

We have 12 unknown pit variables: P₁,₃, P₂,₂, P₂,₃, P₃,₁, P₃,₂, P₃,₃, P₄,₁, P₄,₂, P₄,₃, P₄,₄, and others.

Each can be True or False, so:

2¹² = 4,096 possible configurations

The Algorithm

  1. For each of the 4,096 configurations:
    1. Check: Does this configuration produce the breezes we observed?
    2. If yes, compute the probability of this configuration
    3. If no, skip it
  2. Sum the probabilities for configurations where [1,3] has a pit → P([1,3] has pit | evidence)
  3. Repeat for [2,2] and [3,1]

Does It Work?

Yes! This method is correct. It's called exact inference.

What's the Problem?

4,096 configurations is a lot for a small 4×4 grid. In a real-world probabilistic system with many variables, enumeration becomes infeasible. A 10×10 grid would have 2¹⁰⁰ configurations — more than atoms in the universe!

Computational Explosion: Enumeration works for toy problems but fails in real applications. We need smarter methods. That's where conditional independence comes in.

5. The Key Insight: Conditional Independence

The Breakthrough Idea

Here's a crucial observation: once we fix the pit values for the "frontier" squares (those adjacent to visited squares), the breezes at visited squares become independent of the "far away" squares.

The Independence Insight: The breezes we feel depend only on pits in nearby squares. Distant pits don't affect the breeze we feel today. So we can ignore the distant pits when computing probabilities about nearby pits!

How We Use This

Let's partition the unknown squares into groups:

Frontier: Squares adjacent to visited squares

  • [1,3], [2,2] — can affect breezes at visited squares
  • [3,1] — can affect breezes at visited squares

Other: All remaining unknown squares

  • Squares like [2,3], [3,2], [3,3], [2,4], etc.
  • These are too far away to affect the breezes we observed

The Mathematical Trick

We can factor out the "Other" squares because they don't affect the observed breezes:

P(Frontier | Evidence) ∝ P(Evidence | Frontier) × Σ_Other P(Other)

The sum Σ P(Other) over all configurations of "Other" squares equals 1 (it's a probability distribution). So it factors out! This means:

P(Frontier | Evidence) ∝ P(Evidence | Frontier)

What This Saves Us

Instead of enumerating 2¹² = 4,096 configurations, we only enumerate the Frontier:

3 frontier variables (P₁,₃, P₂,₂, P₃,₁)

2³ = 8 configurations

We reduced the problem from 4,096 to just 8! From 2¹² to 2³. This is the power of conditional independence.

6. The Five Consistent Models & Their Probabilities

What Are Consistent Models?

A model is a specification of which squares have pits. A model is consistent if it produces exactly the breezes we observed: breezes at [1,1], [1,2], and [2,1], and no breeze where not observed.

Let's check all 8 frontier configurations:

P₂,₂ = true, P₁,₃ = false, P₃,₁ = false

Normalizing Probabilities

After checking all 8 configurations, we sum the probabilities of consistent models and normalize. This gives us:

Square P(Pit | Evidence) Danger Level
[1,3] ≈ 0.31 Medium Risk
[2,2] ≈ 0.86 Very Dangerous!
[3,1] ≈ 0.31 Medium Risk
The Key Numbers:
  • [1,3]: 31% chance of pit
  • [2,2]: 86% chance of pit (nearly certain!)
  • [3,1]: 31% chance of pit

7. Logic vs. Probability: A Crucial Difference

What the Logical Agent Knows

From Chapter 7, a logical agent using knowledge representation and inference would conclude:

  • Pit(2,2) ∨ Pit(1,3) ∨ Pit(3,1) (at least one of these has a pit)
  • But all three are potentially safe squares

The logical agent essentially says: "I have no preference. Pick any of the three."

What the Probabilistic Agent Knows

Our probabilistic agent can rank them by safety:

  • [1,3]: Safest (69% safe)
  • [3,1]: Equally safe (69% safe)
  • [2,2]: Very dangerous (only 14% safe!)

Why Does Probability Give More Information?

Logic only tracks what's possible. Probability assigns degrees of belief to possibilities. Degrees of belief are richer information:

  • Logic: "It might rain tomorrow" (binary: possible or impossible)
  • Probability: "There's a 70% chance of rain tomorrow" (degrees: very likely, somewhat likely, unlikely, etc.)

The Insight: Both agents have the same observations (breezes). The logical agent says "anything is possible." The probabilistic agent says "some things are much more likely than others." That extra information makes better decisions possible.

8. Connecting to the Bigger Picture

Three Big Ideas in This Example

Independence

Each square independently has a 20% chance of containing a pit. The pit at [2,2] doesn't influence the probability of a pit at [3,3]. This independence simplifies our probability model — without it, we'd need to specify relationships between every pair of squares.

Conditional Independence

Given the pit locations, breezes are independent of distant pits. This is why we could factor out the "Other" squares. Conditional independence lets us decompose a hard problem into easier subproblems — in this case, from 4,096 configurations to 8.

Bayesian Inference

We combined prior knowledge (prior probability of pits) with observed evidence (breezes) to compute posterior probabilities (probability given evidence). This is Bayes' rule in action: asking "what's the probability of pits given what I've observed?"

Why Does This Matter?

The Wumpus World is a toy problem, but these three ideas are used everywhere:

  • Medical Diagnosis: "Patient has symptom X. What's the probability of disease Y?" (Bayesian inference)
  • Spam Detection: "This email has certain words. Likely spam or legitimate?" (Conditional independence helps: we treat words independently)
  • Robotics: "Robot sensed obstacle. How confident?" (Combining sensor independence with evidence)
  • Weather Prediction: "Current conditions suggest..." (Bayesian networks use conditional independence)
The Principle: Many real-world systems have independence structure (some things don't affect others) or conditional independence (things become independent once you know key information). Exploiting this structure is how probabilistic AI scales from toy problems to real applications.

Summary: What You've Learned

  • The Wumpus World is a 4×4 grid where an agent explores, facing hidden dangers (pits) revealed by clues (breezes).
  • The Problem: Given observed breezes, which unexplored square is safest? Logic can't answer this; probability can.
  • The Probability Model: Random variables for pits (P_ij) and breezes (B_ij), with a full joint distribution combining prior probabilities and deterministic breeze rules.
  • Naive Approach: Enumerate all 2^12 = 4,096 pit configurations. It works but is computationally expensive.
  • The Key Insight — Conditional Independence: Distant pits don't affect observed breezes, so they factor out. This reduces 4,096 configurations to just 8 frontier configurations.
  • The Results:
    • [1,3]: 31% chance of pit (safe to visit)
    • [2,2]: 86% chance of pit (very dangerous!)
    • [3,1]: 31% chance of pit (safe to visit)
  • Logic vs. Probability: A logical agent can't distinguish between the three squares. A probabilistic agent knows [2,2] is far more dangerous. Probability gives us degrees of belief, not just binary possibilities.
  • Broader Impact: This example combines independence, conditional independence, and Bayesian inference — three core ideas from Chapter 12 — to solve a realistic problem. These principles scale to medical diagnosis, spam detection, robotics, and more.

The Big Takeaway: Probability lets us quantify uncertainty and make better decisions than pure logic can. By understanding independence and conditional independence, we can even make sophisticated probabilistic reasoning tractable on real problems.

Key Formulas to Remember

Full Joint Distribution: P(P₁,₁, ..., P₄,₄, B₁,₁, ..., B₄,₄) = P(B|P) · ∏ P(Pᵢ,ⱼ)
Bayes' Rule: P(Query | Evidence) = P(Evidence | Query) · P(Query) / P(Evidence)
Conditional Independence: P(X | Y, Z) = P(X | Z) when X ⊥ Y | Z (X is independent of Y given Z)
Prior Probability: P(Pit at [i,j]) = 0.2 for all [i,j] except [1,1]
